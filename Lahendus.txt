Käivitamise õpetus:

1) Lae alla ja installi XAMPP (Sellega saad MySQL ja PHP kasutada oma arvuti peal)
2) Ava XAMPP ja aktiveeri Apache ja MySQL
3) Sinu server on C://xampp/htdocs kaustas, sinna saad ligi URLiga http://localhost/.
4) Kõik 4 faili peavad olema sama kausta sees
5) Ava XAMPP-is MySQL -> Admin, sellega kuvad PHPMyAdmin lehe.
Tee uus andmebaas vajutades nupule "database", alla sellele nimi "concretedb".
6) Kliki "SQL" tab-i peale ning käivita järgnevad laused:

CREATE TABLE concreteProducts (
	productID int NOT NULL AUTO_INCREMENT,
	productName VARCHAR(255),
	productWeight int NOT NULL,
	transported BOOL DEFAULT FALSE,
	PRIMARY KEY (productID)
);

CREATE TABLE trucks (
	truckID int NOT NULL AUTO_INCREMENT,
	truckName VARCHAR(255),
	maxLoad int NOT NULL,
	currentLoad int NOT NULL,
	money int NOT NULL,
	PRIMARY KEY (truckID)
);

CREATE TABLE loadedProducts (
	truckID int NOT NULL,
	productID int NOT NULL,
	FOREIGN KEY (truckID) REFERENCES trucks(truckID),
	FOREIGN KEY (productID) REFERENCES concreteProducts(productID)
);

7) Nüüd on sul olemas töötav andmebaas. Võid käivitada setup.php faili, ava see localhostis.
Setup faili käivitades lisatakse 100 tonni väärtuses suvalise 55-5555kg suuruseid betoon produkte.
8) Võid avada index.php ning lasta kaubaautodel läbi minna. Kaubaauto max laadung on vahemikus 1000-8000 kg.
Edukalt läbitud kaubaautode kohta tekib andmebaasis uusi ridu ning seda avaldatakse index.php-l.
9) Setup faili uuesti käivitades kustutatakse kõigis tabelites kõik read ning genereeritakse uus jada produkte.
Taaskasutatavus ftw.








Informatsioon:

Alustasin laupäeval, kuna K-N-R olid veidike kõrvaltegevusi täis.
Aga vähemalt jõudsin selle nädalaga ära tehtud, nagu lubasin.

Aega võttis umbes 5-6 tundi, nii et jäin ikkagi suhteliselt ajapiiri sisse.

Ülesanne oli päris sobilik minu jaoks. Mul oli selline hea positiivne reaktsioon kui andmebaas sai üles seatud
ning PHP ja MySQL vaheline suhtus hakkas tööle. Kõige lahedam oli muidugi näha, et kõik betoonkaubad on
ilusti andmebaasis genereeritud ning kaubaautode läbi sõitmine lisas andmebaasi juurde, mis auto mis kaubad
ära viis.

Raskused:
- See polnud küll eriti raskus, aga hea on veidike PHP oskustelt rooste maha kraapida ning jälle järje peale tagasi saada.
- MySQLi vormis vastuse parsimine tekitas veidike mõtisklusi, et kuidas kõige paremini info kiiresti ja efektiivselt
kätte saada ja kasutada
...Ülejäänud ei olnud raske, aga lihtsalt vajas veidike taasmeenutust.

Lisa/Optimeerimine:
- Kasutajaliidest saaks muidugi palju paremaks teha, aga ma tegin selle praegu nii lihtsaks kui võimalik.
Saaks muidugi linkida paar lisalehte, et kätte saada setup.php ning ka leht, kus saab taas kontrollida, mis kaubik
milliseid kaupu peaks vastu võtma. Praegu annab see vaid tagasiside pealehel üks kord. Aga andmebaasis on
info olemas, vaja on lihtsalt uus leht, SQL lause(d) ning echo/print, aga ma ei taha liiga palju aega sellele veeta.
See SQL lause vajab ilmselt palju INNER/LEFT JOIN-e, millega läheks mull lisaks +1 tund arvatavasti.
Ma üritan ikkagi ajalimiidist mingil määral kinni pidada.
- Tehniliselt ei oleks "money" ja "currentLoad" väärtuseid andmebaasis vaja, need saaks relatsioonide kaudu ka kätte,
aga ma lisasin need lihtsuse pärast, sest eelmises punktis mainitud lisalehte ei ole ning INNER JOIN SQL lausete
tegemine mind hetkel just eriti ei erutanud
- SQL lausete saatmine ning vastuste parsimine PHP-s. Seda saaks arvatavasti teha veidike optimeeritumalt/puhtamale,
aga ma pole hetkel päris kindel kuidas, nii et praegu on nad lihtsalt nii nagu nad on.
$printResult stringi/html ehitamist saaks ka arvatavasti puhtamalt teha kui see mida ma tegin
- Betoonkaupu saaks veel optimeeritumalt laduda, kuid ma ei tahtnud pead vaevata rekursiivsete algoritmise tegemisega
PHPs nii et ma jätsin selle praegu väga lihtsaks.

Aga vähemalt kõik töötab, informatsioon on kõik olemas, ning kood on relatiivselt lihtne, lühike ja arusaadav.
Ma ei kasutanud küll mingeid frameworke vms, sest et ma ei usalda hetkel, et ninde paigaldamine ja kasutamine
veel lisatunde juurde ei lisa, aga nendega tahakski ma lähitulevikus lähemalt tutvuda, et neid kasutada ning koodi
paremini optimeerida ja puhtana hoida jms.
